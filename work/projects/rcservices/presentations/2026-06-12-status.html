Status report · June 12, 2026 · covering the last 30 days
Prepared by Eli Luvish — Research Computing
RC Services is steadily bringing storage billing for three services — MAD3, BriefCase, and RFA — fully in-house, taking it over from a long-standing manual arrangement and moving it off aging infrastructure. RFA usage is now collected automatically; the BriefCase and MAD3 pipelines are built and entering a careful side-by-side validation period before cutover. Separately, the FreezerPro billing integration cleared its main blocker and is ready to build once access is provided. A few decisions need stakeholder input — see below.
Recently shipped

What's new for users

  • RC Services now collects RFA storage usage automatically every day, so it can be billed without anyone running the old manual scripts. [#1994]
  • A new My Applications menu gives users one place to launch the services available to them. [#1992]
  • Cancel-and-refund is now available for pre-billed software services, so a service paid for in advance can be cancelled and refunded cleanly. [#1954]
  • Subscriptions can no longer be placed against an expired fund — the system blocks it up front instead of letting a charge land on a closed grant. [#1983]
  • Adding a new fund mid-project is now a cleaner picker and no longer breaks the page. [#1959, #1953]
  • You can now preview the full body of a sent email in a pop-up window, without leaving the page. [#1956]
  • The project edit page is organized into tabs, making fields easier to find. [#1946]
  • The description editor now has a formatting toolbar and shows formatted text consistently. [#1978]
  • Fixed: a project no longer loses its owner when the account field is left blank. [#1986]
In progress

What's being built

  • Storage billing takeover (MAD3, BriefCase, RFA). The three storage services are being moved onto one modern, automated billing pipeline inside RC Services, replacing scripts that have run for years on aging equipment. RFA collection is live; MAD3 and BriefCase collection are built and being validated against the existing numbers before any switch.
  • FreezerPro billing integration. The data and billing approach are now agreed with the REDCap team — annual, tier-based billing pulled once a month. Building begins as soon as the REDCap team provides the access credentials. [#1844]
Decisions needed

Where we'd like your input

How should monthly storage charges be calculated from daily usage readings?

The new system records one usage reading per day; monthly billing needs a single figure. Recommendation: match the current method (75th percentile) during the transition so the side-by-side comparison stays clean, and revisit only if it clearly over- or under-bills.

How should FreezerPro cancellations be handled?

A cancellation approach was agreed in an earlier discussion but never written down. We need to re-confirm what happens when a lab cancels FreezerPro mid-cycle. Low-frequency, but worth pinning down before launch.

Should grant financial contacts get self-service visibility into a grant's subscriptions?

Financial contacts often reach out after invoices land. Giving them visibility into all of a grant's subscriptions could let them self-serve on invoice questions rather than routing everything through the analyst team. For Alissa's call.

Risks & blockers

What could slow us down

  • Aging equipment behind current storage billing. Today's billing still leans on older equipment that has become less reliable. The takeover removes that dependency — completing the cutover is what fully retires the risk.
  • FreezerPro is waiting on access credentials from the REDCap team before the integration can be built. Everything else is ready.
  • A handful of storage volumes can't yet be matched to a billing account. These are being reconciled with MGB Digital Storage so nothing is billed incorrectly or missed.
What's next

Near-term plan

  • Wire the new RFA usage collection into the monthly billing run.
  • Run the new MAD3 and BriefCase billing side-by-side with the existing numbers through June and July, reconcile any differences, and aim for a full switch-over in August.
  • Build the FreezerPro integration as soon as the REDCap access credentials arrive.
  • Confirm the open decisions above so the billing methodology and cancellation handling are settled before cutover.
